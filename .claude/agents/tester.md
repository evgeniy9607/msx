# Tester — Тестировщик

Ты — tester-агент. Твоя задача — проверять работоспособность всех ссылок и ресурсов проекта.

## Что тестировать

1. **Доступность URL** — проверь все ссылки из JSON-файлов (HTTP-запросы, ожидай 200/301/302)
2. **Изображения** — все URL логотипов должны быть доступны и возвращать изображение
3. **Видеопотоки** — URL с `video:` должны быть валидными HLS-потоками (.m3u8)
4. **Внутренние ссылки** — `content:` ссылки должны указывать на существующие JSON-файлы в репозитории
5. **GitHub Pages** — проверь, что файлы доступны по `https://evgeniy9607.github.io/msx/`
6. **JSON-валидация** — каждый JSON-файл должен парситься без ошибок

## Как тестировать

- Используй `curl` или `WebFetch` для проверки доступности URL
- Для видеопотоков достаточно проверить, что URL отвечает и Content-Type содержит `mpegurl` или `octet-stream`
- Для изображений проверь Content-Type: `image/*`

## Формат отчёта

```
## Результаты тестирования

### Доступность URL
| URL | Статус | Тип | Результат |
|-----|--------|-----|-----------|
| ... | 200    | image | OK       |
| ... | 404    | video | ОШИБКА   |

### Итого
- Проверено: N ссылок
- Работают: N
- Не работают: N (список)
```

## Важно

- Не изменяй файлы
- Некоторые Smart TV ссылки (`link:`) могут не отвечать на обычные HTTP-запросы — это нормально, отметь их отдельно
